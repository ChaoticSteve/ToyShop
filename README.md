# ToyShop

## Описание проекта

Консольное Java-приложение, моделирующее систему розыгрыша игрушек на основе заданной вероятности (веса).

Каждая игрушка имеет частоту выпадения, на основе которой формируется очередь с приоритетом. При извлечении из очереди выбираются игрушки с наибольшей частотой.

Результаты розыгрыша сохраняются в файл.

---

## Цель проекта

Проект разработан для демонстрации:

* работы с коллекциями (PriorityQueue)
* применения интерфейсов
* использования паттерна Factory
* разделения логики по слоям

---

## Функциональные возможности

* создание объектов игрушек через фабрику
* хранение игрушек в приоритетной очереди
* выбор игрушек на основе их "веса" (частоты)

* обработка ситуации пустой очереди

## Используемые технологии

## Архитектура проекта

Проект построен с использованием интерфейсов и разделения ответственности.

### Основные компоненты:

#### Интерфейсы

* `iToy` — контракт для объекта игрушки 
* `iToyCollection` — управление коллекцией игрушек 
* `iToyFactory` — создание объектов игрушек 

---

#### Реализации

**Toy**

* модель игрушки (id, название, частота) 

**ToyFactory**

* создание экземпляров игрушек (паттерн Factory) 

**ToyCollection**

* хранение игрушек в PriorityQueue
* выбор игрушек по приоритету (частоте)
* запись результатов в файл 

---

#### Точка входа

**Main**

* создание набора игрушек
* запуск процесса выборки
* запись результатов в файл 

---

## Принцип работы

1. Создаются игрушки с параметрами:

   * id
   * название
   * частота выпадения

2. Игрушки помещаются в приоритетную очередь, где:

   * чем выше частота, тем выше приоритет

3. При вызове получения игрушки:

   * выбирается элемент с максимальной частотой

4. Результаты записываются в файл `output.txt`

---

## Пример данных

Игрушки:

* конструктор (частота 2)
* робот (частота 2)
* кукла (частота 6)

При извлечении чаще будет выпадать "кукла", так как имеет наибольший приоритет.

---

## Как запустить

1. Скомпилировать проект:

```bash
javac Main.java
```

2. Запустить:

```bash
java Main
```

3. Результаты появятся в файле:

```
output.txt
```

## Ограничения текущей реализации

* приоритет определяется только частотой, без реального вероятностного распределения
* отсутствует случайность (выбор всегда детерминирован)
* нет пользовательского ввода
* простая запись в файл без форматирования
